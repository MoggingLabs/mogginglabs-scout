# Contributing

This repository uses a feature-branch and pull-request workflow.

## Rules

- Do not commit secrets or real prospect data.
- Use synthetic fixtures only.
- Keep changes scoped to linked issues.
- Include tests or explain why a test is not applicable.
- Include security/compliance notes for data, auth, AI, or outreach changes.
- Do not add route planning features; they are explicitly out of scope.

## Pull requests

Every PR should include:

- Summary
- Linked issue
- Test plan
- Screenshots for UI changes
- Security/compliance notes
- Data/secrets impact
