# Security Policy

## Reporting security issues

Report security concerns privately to the repository owner or MoggingLabs maintainers. Do not open public issues for vulnerabilities, leaked secrets, or private data exposure.

## Data policy

This public repository must not contain:

- real lead/prospect data
- real contact lists
- API keys or credentials
- Supabase service role keys
- Vercel tokens
- AI provider keys
- OAuth secrets
- screenshots containing real prospects

Use synthetic data only.
